# 블록 퍼즐 하위 장르 분석 보고서

> 작성일: 2026-02-22
> 목적: 블록 퍼즐 장르 내 세부 타입을 분류하고, 프로젝트에 구현 가능한 타입을 선정

---

## 1. 블록 퍼즐의 기원

모든 블록 퍼즐 게임은 **테트리스의 3가지 핵심 규칙**에서 파생되었다:

1. 공간 효율적으로 피스 맞추기
2. 빈 공간 피하기
3. 보드 생존 유지하기

이 규칙을 현대 모바일 캐주얼 퍼즐 시장에 맞게 재해석한 것이 현재의 블록 퍼즐 장르다.

---

## 2. 블록 퍼즐 세대별 진화

Deconstructor of Fun의 분석(2026.01)에 따른 3세대 분류:

### 1세대: 단순 제약 + 무한 진행
- 테트리스 메카닉을 게임화한 형태
- 턴 기반으로 3개 피스를 선택하여 배치
- 광고 수익 중심 (IAP 없음)
- **대표작**: Block Blast, 1010!

### 2세대: 설계된 보드 + 목표 시스템
- 정적 레벨 구조 도입, 보드 자체가 하나의 퍼즐
- 색상 게이트 등으로 미니 퍼즐 형성
- IAP 친화적 (부스터, 세이브)
- **대표작**: Color Block Jam

### 3세대: 테마 차별화 + 고급 메카닉
- 다양한 테마(자동차, 동물, 화살표 등) 적용
- 클래식 퍼즐 요소(장애물, 부스터) 적극 활용
- 높은 LTV의 IAP 수익 모델
- **대표작**: Car Jam, Bus Escape, Gecko Out

## 3. 구현 관점의 4가지 타입 분류

### 타입 A: 클래식 낙하형 (테트리스)

```
┌──────────┐
│          │  ← 블록이 위에서 떨어짐
│    ██    │
│    ██    │  실시간 회전/이동
│  ████    │
│████████  │  줄 완성 시 제거
└──────────┘
```

| 항목 | 내용 |
|---|---|
| 핵심 메카닉 | 실시간 블록 낙하, 회전, 수평 이동 |
| 조작 | 키보드 / 터치 스와이프 + 탭 회전 |
| 시간 압박 | **있음** (점점 빨라지는 낙하 속도) |
| 게임 오버 | 블록이 천장에 닿으면 종료 |
| 구현 난이도 | **중상** |
| 주요 구현 요소 | 타이머 루프, 충돌 판정, 회전 매트릭스, 키 입력 처리 |
| 모바일 적합성 | **낮음** — 터치 조작으로 정밀한 회전/이동이 어려움 |
| 리스크 | 테트리스 IP 유사성 문제, 모바일 UX 난이도 |

### 타입 B: 드래그 배치형 (Block Blast / 1010!)

```
┌─────────────┐
│ . . . . . . │
│ . . ■ ■ . . │  격자에 드래그하여 배치
│ . ■ ■ ■ . . │
│ ■ ■ ■ ■ ■ ■ │  ← 줄 완성! 제거됨
│ . ■ . ■ ■ . │
└─────────────┘
   [██] [■] [██■]  ← 매 턴 3개 블록 제공
```

| 항목 | 내용 |
|---|---|
| 핵심 메카닉 | 매 턴 주어진 블록 조각을 격자에 드래그하여 배치 |
| 조작 | 드래그 앤 드롭 (터치 최적화) |
| 시간 압박 | **없음** — 원하는 만큼 고민 가능 |
| 게임 오버 | 3개 블록 중 어느 것도 배치할 수 없으면 종료 |
| 격자 크기 | 8x8 (Block Blast) / 10x10 (1010!) |
| 구현 난이도 | **중** |
| 주요 구현 요소 | 드래그앤드롭, 배치 가능 판정, 줄 완성 판정, 블록 조각 생성 |
| 모바일 적합성 | **최적** — 한 손 드래그 조작, 시간 압박 없음 |
| 성과 | 2025년 전 세계 다운로드 **1위** (Block Blast 3.56억) |

### 타입 C: 스도쿠 혼합형 (Woodoku)

```
┌───┬───┬───┐
│ . ■│■ .│. .│
│ . .│■ .│. .│  9x9 격자 + 3x3 영역
│ ■ ■│■ .│. .│
├───┼───┼───┤
│ . .│. .│. .│  가로줄 / 세로줄 / 3x3 블록
│ . .│. .│. .│  완성 시 제거
│ . .│. .│. .│
├───┼───┼───┤
│ . .│. .│. .│
│ . .│. .│. .│
│ . .│. .│. .│
└───┴───┴───┘
```

| 항목 | 내용 |
|---|---|
| 핵심 메카닉 | 블록 배치 + 가로/세로/3x3 영역 완성 시 제거 |
| 조작 | 드래그 앤 드롭 (타입 B와 동일) |
| 시간 압박 | **없음** |
| 게임 오버 | 블록 배치 불가 시 종료 |
| 격자 크기 | 9x9 (3x3 서브그리드 9개) |
| 구현 난이도 | **중** (타입 B + 3x3 영역 판정) |
| 주요 구현 요소 | 타입 B 전부 + 3x3 영역 완성 판정 로직 |
| 모바일 적합성 | **좋음** — 타입 B와 동일한 UX |
| 차별점 | "블록 퍼즐 + 스도쿠" 브랜딩으로 전략성 어필 |

### 타입 D: 레벨 설계형 (Color Block Jam)

```
  레벨 17         목표: 빨강 3개 제거
┌─────────────┐
│ 🟥 🟦 🟦 . │
│ 🟥 🟨 . . . │  미리 디자인된 퍼즐
│ 🟥 🟨 🟨 🟦 │
│ . . 🟦 🟦 . │  정해진 목표 달성 시 클리어
└─────────────┘
```

| 항목 | 내용 |
|---|---|
| 핵심 메카닉 | 사전 디자인된 보드에서 목표 달성 |
| 조작 | 탭 / 드래그 (게임마다 다름) |
| 시간 압박 | 없음 (수 제한은 있을 수 있음) |
| 게임 오버 | 목표 수 초과 시 실패 |
| 구현 난이도 | **상** |
| 주요 구현 요소 | 레벨 에디터/생성기, 목표 시스템, 수백 개 레벨 데이터 |
| 모바일 적합성 | 좋음 |
| 리스크 | **콘텐츠 볼륨 문제** — 레벨이 적으면 금방 소진됨 |

## 5. 핵심 구현 요소 상세 (타입 B 기준)

타입 B(Block Blast류)를 구현한다고 가정할 때 필요한 핵심 모듈:

### 5.1 게임 로직 (`block-puzzle.ts`)
```
- 격자 상태 관리 (8x8 or 10x10 2D 배열)
- 블록 조각 정의 (테트로미노 + 변형 셰이프)
- 배치 가능 판정 (격자 범위 + 빈칸 확인)
- 줄 완성 판정 (가로/세로 풀 라인 스캔)
- 줄 제거 + 점수 계산 (콤보 보너스)
- 게임 오버 판정 (3개 블록 모두 배치 불가)
- 블록 조각 랜덤 생성기
```

### 5.2 보드 컴포넌트 (`BlockPuzzleBoard.tsx`)
```
- NxN 격자 렌더링 (Emotion styled)
- 드래그 앤 드롭 (터치 + 마우스)
- 배치 미리보기 (드래그 중 유효 위치 하이라이트)
- 줄 제거 애니메이션 (Framer Motion)
- 블록 조각 트레이 (하단 3개 표시)
```

### 5.3 점수 시스템
```
- 현재 점수 / 하이스코어
- 콤보 보너스 (동시 여러 줄 제거)
- 연속 콤보 (연속 턴 줄 제거 시 배율)
```

### 5.4 Woodoku 확장 시 추가 요소
```
- 격자 크기 변경 (9x9)
- 3x3 서브그리드 시각적 구분선
- 3x3 영역 완성 판정 로직 추가
- 점수 체계에 영역 클리어 보너스 추가
```

## 6. 추천 구현 전략

### Phase 1: 타입 B (Block Blast류) 구현
- 8x8 격자 + 드래그 배치 + 줄 완성/제거
- 블록 조각 랜덤 생성 + 점수/하이스코어
- 게임 오버 + 재시작 플로우
- **예상 작업**: 게임 로직 + 보드 컴포넌트 + 점수 시스템

### Phase 2 (선택): 타입 C (Woodoku) 모드 추가
- 9x9 격자 + 3x3 서브그리드 판정 추가
- 모드 선택 UI (Block Blast / Woodoku)
- 별도 하이스코어 트래킹

### Phase 3 (선택): 부가 기능
- 일일 도전 (시드 기반 동일 블록 시퀀스)
- Supabase에 하이스코어 저장 + 랭킹
