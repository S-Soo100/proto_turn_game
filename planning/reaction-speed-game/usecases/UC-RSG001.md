## 사전 조건

- 플레이어가 로그인 상태
- 게임 선택 화면에 진입한 상태

## 기본 시나리오 (Happy Path)

1. 플레이어가 "시작" 버튼을 누른다
2. 상단 HUD에 120초 타이머가 시작된다
3. 스케줄에 따라 게임 영역에 타겟 서클이 등장한다 (바깥 원이 줄어드는 축소 애니메이션)
4. 플레이어가 타겟 서클을 클릭한다
5. 시스템이 소멸까지 남은 시간을 계산하여 등급(Perfect/Great/Good/OK)과 점수를 판정한다
6. 하단 상태바에 점수가 증가하고, 콤보 카운트가 올라간다 (5연속 이상 시 배율 표시)
7. 타겟 서클 위치에 등급 텍스트 + 점수 팝업이 잠깐 표시된다
8. 3~7이 120초 동안 반복된다
9. 120초가 종료되면 게임이 자동으로 멈추고, 결과 화면이 표시된다

## 대안 시나리오

### A1: 타겟 서클 미스 (소멸)
- 조건: 플레이어가 클릭하지 못한 채 타겟 서클의 duration이 끝남
- 흐름:
  1. 타겟 서클이 "MISS" 텍스트와 함께 페이드아웃
  2. 콤보 카운트가 0으로 초기화
  3. 미스 카운트가 1 증가
  4. 점수 변동 없음 (감점 없음)

### A2: Speed 타겟 서클 등장
- 조건: 스케줄에 Speed 타입 타겟이 등장
- 흐름:
  1. 빨간색 타겟 서클이 등장 (Normal과 시각적 구분)
  2. 2초 안에 클릭해야 함 (Normal의 절반 시간)
  3. 클릭 성공 시 Normal 대비 1.5배 점수

### A3: 동시 다수 타겟
- 조건: 이전 타겟이 아직 활성화된 상태에서 새 타겟 등장
- 흐름:
  1. 화면에 여러 타겟 서클이 동시에 표시됨
  2. 플레이어는 원하는 순서로 클릭 가능
  3. 각 타겟은 독립적으로 소멸 타이머 진행

## 예외 시나리오

### E1: 게임 영역 외부 클릭
- 조건: 플레이어가 타겟이 없는 빈 영역을 클릭
- 시스템 반응: 아무 반응 없음 (페널티 없음, 콤보 유지)

### E2: 브라우저 백그라운드 전환
- 조건: 게임 중 다른 탭으로 전환하거나 화면을 벗어남
- 시스템 반응: 타이머는 계속 진행 (일시정지 없음, 공정성)

## 검증 방법

- [ ] 시작 → 120초 플레이 → 자동 종료 → 결과 화면 전체 흐름 재현
- [ ] Normal 타겟 클릭 시 등급 + 점수가 올바르게 표시
- [ ] Speed 타겟 클릭 시 1.5배 점수 적용 확인
- [ ] 5/10/20/30 연속 클릭 시 콤보 배율 x2/x3/x4/x5 확인
- [ ] 미스 시 콤보 초기화 + MISS 표시 확인
- [ ] 동시 다수 타겟 상황에서 각각 독립적으로 동작 확인
