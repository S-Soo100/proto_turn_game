## 요약

> 개발 환경(`import.meta.env.DEV`)에서만 동작하는 디버그 패널을 공기놀이에 추가하여, 변칙 룰 강제 발동 · 확률 오버라이드 · 상태 로그 확인을 가능하게 한다. 프로덕션 빌드에는 포함되지 않는다.

## 배경

현재 변칙 룰을 테스트하려면:
- Unit 테스트로 로직만 검증 가능
- 실제 게임에서 확률 의존적으로 발동 → 특정 룰을 보려면 운에 의존
- 발동 시 콘솔 로그 없음 → 내부 상태 확인 불가

개발/QA 시 **특정 변칙 룰을 즉시 확인**하는 수단이 필요하다.

## 상세 설명

### 1. 디버그 패널 UI

게임 화면 좌하단에 `🐛` 버튼 → 탭 시 디버그 패널 토글.

패널 내용:
| 섹션 | 기능 |
|---|---|
| **강제 발동** | 7개 변칙 룰 버튼 (탭 → 해당 룰 즉시 발동) |
| **확률 오버라이드** | 슬라이더 0~100% (기본값: `auto` = 원래 확률 커브 사용) |
| **상태 모니터** | round, stage, substep, phase, 발동된 chaos 목록 실시간 표시 |

### 2. 콘솔 로그

`import.meta.env.DEV`일 때 chaos 발동 시 자동 로그:
```
[CHAOS] rule=cat-swipe trigger=before-success round=1 stage=3
```

### 3. URL 파라미터 (선택)

`/gonggi?chaos=cat-swipe` → 첫 번째 chaos 체크 시 해당 룰 강제 발동.
`/gonggi?chaosChance=1.0` → 확률 100%로 오버라이드.

### 4. 프로덕션 안전성

- 모든 디버그 코드는 `import.meta.env.DEV` 가드 내부
- Vite tree-shaking에 의해 프로덕션 빌드에서 완전 제거
- 디버그 패널 컴포넌트는 `React.lazy` + 조건부 import

### GonggiBoard 내 디버그 통합

```tsx
// DEV-only state
const [debugForceRule, setDebugForceRule] = useState<string | null>(null)
const [debugChanceOverride, setDebugChanceOverride] = useState<number | null>(null)

// chaos 체크 시
const chaos = checkChaos(
  state,
  trigger,
  ALL_CHAOS_RULES,
  rngRef.current,
  import.meta.env.DEV ? debugForceRule : undefined,      // forceRuleId
  import.meta.env.DEV ? debugChanceOverride : undefined,  // overrideChance
)

// 발동 로그
if (import.meta.env.DEV && chaos) {
  console.log(`[CHAOS] rule=${chaos.rule.id} trigger=${trigger} round=${state.round} stage=${state.stage}`)
}

// 패널 렌더링
{import.meta.env.DEV && (
  <GonggiDebugPanel
    gameState={gameState}
    onForceRule={setDebugForceRule}
    onSetChanceOverride={setDebugChanceOverride}
    chanceOverride={debugChanceOverride}
  />
)}
```

### gonggi-chaos.ts 변경

```typescript
export function checkChaos(
  state: GonggiState,
  trigger: ChaosTrigger,
  rules: ChaosRule[],
  rng: () => number,
  forceRuleId?: string | null,     // NEW
  overrideChance?: number | null,  // NEW
): { result: ChaosResult; rule: ChaosRule } | null {
  // Force mode: skip probability, find rule directly
  if (forceRuleId) {
    const rule = rules.find(r => r.id === forceRuleId)
    if (rule) {
      const result = rule.execute(state, rng)
      return { result, rule }
    }
    return null
  }

  // Override chance
  if (!shouldTriggerChaos(state.round, rng, overrideChance)) return null
  // ... rest unchanged
}
```

## 테스트 방침

- **Integration 테스트**: 디버그 패널은 DEV 전용이므로 별도 테스트 불필요 (수동 QA)

## 메모

- 디버그 패널 스타일: 반투명 다크 배경, 게임 플레이를 가리지 않도록 좌하단에 접이식 배치
- `forceRuleId`는 한 번 사용 후 자동 초기화 (null로 리셋) → 매번 수동 탭 필요
- URL 파라미터 지원은 선택 사항 — 패널만으로 충분하면 생략 가능
