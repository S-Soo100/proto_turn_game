## 요약

> 던진 돌의 z축 포물선을 CSS animation으로 구현하여 "위로 올라갔다 내려오는" 궤적을 시각적으로 표현하고, 잡기(catch)에 타이밍 요소를 추가한다.

## 상세 설명

### z축 포물선 — 돌이 올라갔다 내려오는 과정

실제 공기놀이에서 돌을 위로 던지면:
1. 손에서 출발 (z=0.3) → 위로 가속
2. 최고점에서 멈춤 (z=1.0)
3. 중력으로 하강 → 바닥 근처에서 잡아야 함 (z≈0)

이를 CSS animation + T-GG009의 z축 시각 시스템으로 표현:

```css
@keyframes toss-arc {
  /* 상승 (ease-out — 올라갈수록 느려짐) */
  0%   { --stone-z: 0.3; }    /* 손 위치 (hold) */
  40%  { --stone-z: 1.0; }    /* 최고점 */

  /* 하강 (ease-in — 내려올수록 빨라짐) */
  45%  { --stone-z: 0.98; }   /* 꼭대기에서 잠시 멈춤 */
  100% { --stone-z: 0.0; }    /* 바닥 도착 */
}
```

CSS 변수 `--stone-z`가 변하면 T-GG009에서 정의한 시각 속성들이 자동 연동:

| z 구간 | 시각 효과 | 플레이어 인식 |
|---|---|---|
| 0.3 → 1.0 (상승) | 돌 작아짐, 위로 이동, 그림자 커짐 | "돌이 올라간다!" |
| 1.0 (정점) | 가장 작은 돌, 가장 큰 그림자, 잠시 정지 | "꼭대기다!" |
| 1.0 → 0.3 (하강) | 돌 커짐, 아래로 이동, 그림자 줄어듦 | "내려온다!" |
| 0.3 → 0.0 (착지 구간) | 돌 원래 크기, 빨간 경고 glow | "잡아야 해!" |

### 단계별 타이밍 테이블

| 단계 | 포물선 duration | 상승 구간 | 정점 체류 | 하강 구간 | pick 가능 시간 | catch 윈도우 |
|---|---|---|---|---|---|---|
| 1단 (일단) | 2.4초 | 0.8초 | 0.2초 | 1.4초 | ~1.2초 | 0.5초 |
| 2단 (이단) | 2.2초 | 0.7초 | 0.2초 | 1.3초 | ~1.0초 | 0.4초 |
| 3단 (삼단) | 2.0초 | 0.7초 | 0.1초 | 1.2초 | ~0.9초 | 0.35초 |
| 4단 (사단) | 1.8초 | 0.6초 | 0.1초 | 1.1초 | ~0.8초 | 0.3초 |
| 5단 (꺾기) | 2.4초 | 0.8초 | 0.2초 | 1.4초 | — | 0.5초 |

- **pick 가능 시간**: 하강 시작 ~ catch 윈도우 직전 (이 동안 바닥 돌 스와이프)
- **catch 윈도우**: z가 0.15~0 구간에서만 잡기 버튼 유효

### 타이밍 기반 잡기

**catch 윈도우 판정**:
- 돌이 z=0.15 아래로 진입 → catch 버튼 활성화 (초록색 glow)
- 돌이 z=0 도달 → 놓침 (자동 실패)
- 이 구간이 catch 윈도우 (0.3~0.5초)

**판정 결과**:
| 타이밍 | 결과 | 피드백 |
|---|---|---|
| catch 윈도우 내 | 성공 | "잡았다!" + 돌 원래 크기 복귀 |
| 너무 빠름 (z > 0.15) | 실패 | "너무 빨라요!" + 돌 떨어짐 |
| 놓침 (z=0 자동) | 실패 | "놓쳤어요!" + 돌 바닥 바운스 |

**시각적 타이밍 가이드**:
- 손 아이콘(🤚) 주변에 TargetCircle 스타일 축소 링
- 링이 손 아이콘 크기와 일치할 때 = catch 윈도우
- 축소 링의 duration = 포물선 하강 시간

### pick 시간 제한

- 하강 시작 시점부터 타이머 시작
- 보드 하단에 타이머 바 표시 (CSS animation, 잔여 시간 시각화)
- 필요 개수를 시간 안에 잡지 못하면 → 자동 실패 (`failSubstep()`)
- 시간 안에 잡으면 → 자동으로 catch phase 진입

### 5단계(꺾기) 특수 처리

- 전체 돌을 위로 던지고 전부 잡는 단계
- select/hold 없이 바로 toss (T-GG010)
- pick 없음 — 바닥에 잡을 돌이 없으므로
- 전체 돌이 z=1.0까지 올라갔다 내려옴
- catch 윈도우에 맞춰 잡기 → 성공/실패

### 포물선 구현 선택지

**Option A: CSS @keyframes + animation (추천)**
```css
@keyframes toss-arc {
  0%   { transform: translateY(0) scale(0.9); filter: drop-shadow(...); }
  40%  { transform: translateY(-80px) scale(0.65); filter: drop-shadow(...); }
  45%  { transform: translateY(-78px) scale(0.66); }
  100% { transform: translateY(0) scale(1.0); filter: drop-shadow(...); }
}
/* animation-duration: stage별 상수 */
```
- 장점: GPU 가속, 타이밍 정확, JS 개입 불필요
- 단점: animation 중간에 상태 읽기 어려움 (catch 타이밍은 JS setTimeout 병행)

**Option B: Framer Motion animate**
```typescript
<motion.div animate={{ y: [0, -80, 0], scale: [0.9, 0.65, 1.0] }} transition={{ duration, times: [0, 0.4, 1] }} />
```
- 장점: 상태 연동 용이, onUpdate 콜백
- 단점: JS 기반이라 성능 약간 열세

→ **추천: CSS @keyframes로 시각, JS setTimeout으로 타이밍 판정** 병행

## 메모

- catch 타이밍 → 변칙 룰 "고양이 습격"(`before-success`)이 catch 성공 직전에 발동하면 더 극적
- 타이밍 실패 시 `failSubstep()` 호출로 기존 실패 흐름 재사용
- CSS @keyframes에서 `--stone-z` custom property를 animate하려면 `@property` 등록 또는 JS 기반 interpolation 필요 — 실제로는 transform/filter 값을 직접 keyframe에 하드코딩하는 게 현실적
- 포물선 중 변칙 룰 발동 시: animation을 pause → 이펙트 표시 → resume 또는 cancel
