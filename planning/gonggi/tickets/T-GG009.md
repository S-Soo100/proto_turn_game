## 요약

> matter.js 물리엔진을 완전히 제거하고, 바닥(x,y 평면) 돌은 게임 로직 좌표 기반 정적 렌더링, 던진 돌은 CSS animation 기반 z축 시뮬레이션으로 전환한다.

## 상세 설명

### 현재 문제

| 현상 | 원인 |
|---|---|
| 돌이 모두 화면 하단에 가라앉음 | matter.js gravity가 항상 작용 → scatter 좌표를 무시 |
| scatter 후 돌이 안 퍼짐 | scatter force가 중력보다 약함 |
| 던진 돌이 10초간 체공 | velocity -10 / gravity 0.001 = 피크 10,000ms |
| 돌 위치가 물리엔진에 종속 | 게임 로직 x,y%가 아닌 물리 좌표만 렌더링에 사용 |

### 공간 모델: x,y 바닥 + z 높이

실제 공기놀이의 공간 구조:
- **바닥 (x,y 평면)**: 돌들이 흩어져 있는 면 → 게임 로직 좌표 그대로 렌더링
- **위로 던지기 (z축)**: 돌이 바닥에서 공중으로 올라감 → CSS로 "높이" 시뮬레이션

```
실제 3D 공간          →    화면 2.5D 표현

   z (높이)                z 높이를 CSS 4단서로 표현:
   ↑  🟡                   ① scale 축소 (멀어짐 = 작아짐)
   │                       ② shadow 확산 (높을수록 그림자 넓음)
   │                       ③ y-offset 감소 (화면에서 위로)
   └───→ x                 ④ opacity 변화 (공중에서 반투명)
  /
 y (바닥)
```

### 변경 방향: matter.js 완전 제거

**이유**:
- 바닥 돌에 중력이 필요 없음 (정적 위치)
- 던진 돌의 포물선은 CSS animation이 더 정확하고 예측 가능
- 변칙 룰 이펙트(cat-swipe, stone-eyes)도 CSS transform으로 대체 가능
- matter.js 의존성 제거로 번들 사이즈 감소

**바닥 돌 (z=0, floor/hand 상태)**:
- 게임 로직의 `stone.x`, `stone.y` (0~100%) → CSS `left`, `top`으로 직접 변환
- scatter 시 랜덤 좌표 그대로 표시
- CSS transition (0.3s ease-out)으로 이동 애니메이션
- 그림자: 바로 아래, 작고 선명

**던진 돌 (z>0, tossed/air 상태)**:
- CSS `@keyframes`로 z축 시뮬레이션 (T-GG011에서 상세 구현)
- 이 티켓에서는 기반 인프라(z 상태 변수, CSS 변수)만 준비

### z축 시각 표현 시스템

```css
/* z값(0~1)에 따른 CSS 변수 */
--stone-z: 0;                                    /* 0=바닥, 1=최고점 */
--stone-scale: calc(1 - var(--stone-z) * 0.35);  /* 1.0 → 0.65 */
--stone-y-offset: calc(var(--stone-z) * -80px);  /* 0 → -80px (위로) */
--shadow-blur: calc(var(--stone-z) * 20px);      /* 0 → 20px */
--shadow-spread: calc(var(--stone-z) * 15px);    /* 0 → 15px */
--shadow-opacity: calc(0.4 - var(--stone-z) * 0.2); /* 0.4 → 0.2 */
```

| z값 | scale | y-offset | 그림자 | 의미 |
|---|---|---|---|---|
| 0.0 | 100% | 0px | 작고 선명 | 바닥에 있음 |
| 0.3 | 90% | -24px | 중간 | 손에서 들어올림 (hold) |
| 0.7 | 76% | -56px | 크고 흐림 | 공중 상승 중 |
| 1.0 | 65% | -80px | 가장 크고 흐림 | 최고점 |

### 변칙 룰 물리 대체

| 변칙 룰 | 현재 (matter.js) | 변경 (CSS) |
|---|---|---|
| cat-swipe | `applyCatSwipeForce()` — Body에 force | CSS `@keyframes` — 돌들이 direction 방향으로 밀려나는 transition |
| stone-eyes | `applyFleeForce()` — 포인터 반대 방향 force | CSS `transform: translate()` — 포인터 위치 기반 계산으로 반대 방향 이동 |
| 기타 (bird, split 등) | 물리 미사용 | 변경 없음 |
