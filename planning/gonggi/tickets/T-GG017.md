## 요약

> 돌을 위로 던진 뒤(toss) 받으려 할 때(catch), 던진 돌이 FlyingStone에서 사라지고 바닥에 다시 나타나서 탭으로 잡을 수 없게 되는 버그를 수정한다.

## 증상

1. toss phase에서 돌을 위로 던짐
2. FlyingStone이 포물선 비행 중 → catch phase 진입
3. FlyingStone을 탭하여 catch 시도
4. **돌이 FlyingStone에서 사라지고 바닥에 다시 렌더링됨** → 잡을 수 없음
5. 또는 catch 성공 후 다음 substep 진입 시 돌 상태가 불일치

## 원인 분석

### Root Cause: `clearTossTimers()`의 `setTossAnimating(false)` 타이밍 문제

`handleCatch()` 실행 순서:

```
1. clearTossTimers()
   → tossTimerRef 클리어
   → setTossAnimating(false)     ← 여기서 FlyingStone 소실 조건 발동
   → setCatchMessage('')

2. catchStone(state, true, 'perfect')
   → stone.status: 'air' → 'hand'
   → advanceSubstep() → 다음 phase 결정

3. setGameState(newState)          ← gameState 갱신은 여기서
```

**문제점**: React 렌더링 시 `tossAnimating=false`와 `gameState` 업데이트가 동시 배칭되더라도, 배칭 전 중간 상태에서 문제가 발생할 수 있음.

### 구체적 시나리오

#### 시나리오 A: chaos 체크 경로 (before-success)
```
handleCatch()
→ clearTossTimers()        → tossAnimating = false
→ checkChaos('before-success') → chaos 발동!
→ handleChaosEffect(...)   → return (early exit)
→ catchStone() 미호출      → stone은 여전히 'air' 상태
→ 렌더링: isAir(true) && tossAnimating(false) → 돌이 보드에 나타남
```
FlyingStone은 `tossAnimating && !isPaused` 조건이 false라서 unmount 됨.
돌은 air 상태이지만 board에서 숨김 조건(`isAir && tossAnimating`)이 false → **보드 바닥에 돌 렌더링**.

#### 시나리오 B: 빠른 연속 탭 (stage 5)
Stage 5에서 모든 돌이 air 상태인데, catch 후 즉시 다음 phase로 전환.
`tossAnimating=false` 시점과 `gameState` 업데이트 사이에 1프레임 gap이 생기면 모든 돌이 보드에 잠깐 나타남.

## 구현 가이드

### 수정 전략: `tossAnimating=false`를 gameState 갱신 이후로 지연

#### 1. `clearTossTimers()`에서 `setTossAnimating` 분리

```typescript
// 수정 전
const clearTossTimers = useCallback(() => {
  tossTimerRef.current.forEach(clearTimeout)
  tossTimerRef.current = []
  setTossAnimating(false)       // ← 이것이 문제
  setCatchMessage('')
}, [])

// 수정 후
const clearTossTimers = useCallback(() => {
  tossTimerRef.current.forEach(clearTimeout)
  tossTimerRef.current = []
  setCatchMessage('')
  // setTossAnimating(false)는 호출하지 않음 → 호출측에서 직접 관리
}, [])
```

#### 2. `handleCatch()`에서 gameState 갱신과 tossAnimating을 동시에 처리

```typescript
const handleCatch = useCallback(() => {
  const state = gameStateRef.current
  if (state.phase !== 'catch') return

  // 타이머만 클리어 (tossAnimating은 아직 유지)
  clearTossTimers()
  if (pickTimerRef.current) clearInterval(pickTimerRef.current)

  // Check before-success chaos
  const bsChaos = checkChaos(...)
  if (bsChaos) {
    // chaos 처리 후에 tossAnimating을 끔
    setTossAnimating(false)
    handleChaosEffect(...)
    return
  }

  const newState = catchStone(state, true, 'perfect')
  // gameState와 tossAnimating을 같은 배칭 내에서 갱신
  setTossAnimating(false)
  setGameState(newState)
  gameStateRef.current = newState
  // ...
}, [])
```

#### 3. auto-miss 타이머도 동일 패턴 적용

`startTossAnimation()`의 autoMiss 콜백에서도 `setTossAnimating(false)`를 `setGameState` 직전으로 이동.

#### 4. 돌 렌더링 안전장치 추가 (방어적 코딩)

```typescript
// 수정 전: air 돌은 tossAnimating일 때만 숨김
const isAir = stone.status === 'air' || stone.status === 'tossed'
if (isAir && tossAnimating) return null

// 수정 후: air/tossed 돌은 항상 보드에서 숨김 (FlyingStone 또는 HandArea가 렌더링)
const isAir = stone.status === 'air' || stone.status === 'tossed'
if (isAir) return null
```
이렇게 하면 `tossAnimating` 상태와 무관하게 air/tossed 돌은 보드에 절대 나타나지 않음.
FlyingStone이 unmount된 후에도 돌이 바닥에 "유령"으로 나타나는 현상 원천 차단.
