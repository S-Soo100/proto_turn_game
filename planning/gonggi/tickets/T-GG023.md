## 요약

> 무한 증식(split) 변칙 룰이 실제 게임에서 발동되지 않는 문제를 점검하고 수정한다.
>
> **결과**: `handleChaosEffect`에서 `SplitEffect` 컴포넌트를 렌더링하지 않고 3초 후 무조건 failSubstep만 실행하고 있었음. SplitEffect를 chaos overlay에 연결하고 `handleSplitSelect`/`handleSplitTimeout` 콜백을 추가하여 3택 인터랙션이 정상 동작하도록 수정.

## 상세 설명

사용자가 플레이 중 무한 증식 룰을 한 번도 본 적이 없다고 보고함. 다음 관점에서 점검이 필요하다:

### 점검 항목

1. **발동 조건 확인**
   - 트리거: `after-toss` — 돌 던진 직후 checkChaos가 정상 호출되는지
   - minRound: 2 — R2 이상에서만 발동하므로 R1에서 끝나는 플레이에서는 절대 안 나옴
   - 같은 트리거(`after-toss`)에 bird-transform(0.25), constellation(0.20)과 경합 — split(0.25)이 선택될 확률 검증

2. **GonggiBoard.tsx에서 split 결과 처리**
   - `handleChaosEffect`에서 `stone-split` 타입 결과를 올바르게 처리하는지
   - SplitEffect 컴포넌트가 정상 렌더링되는지
   - 3택 선택 후 결과(정답/오답) 처리가 정상인지

3. **SplitEffect 컴포넌트**
   - 3개 분열 돌 렌더링
   - 선택 콜백 연결
   - 선택 후 결과 애니메이션

### 가능한 원인 추정

- R2 도달 전에 게임이 끝나는 경우가 많아서 자연적으로 안 나올 수 있음
- `after-toss` 트리거에 3개 룰(bird/split/constellation)이 경합하여 split 선택 확률이 낮을 수 있음
- Board에서 split 결과 핸들링이 누락되어 발동은 되지만 이펙트가 표시되지 않을 수 있음

1. DEV 디버그 패널에서 split 강제 발동하여 이펙트 확인
2. 이펙트가 나오면 → 확률/조건 문제 (minRound, 경합)
3. 이펙트가 안 나오면 → Board 핸들링 또는 컴포넌트 버그
