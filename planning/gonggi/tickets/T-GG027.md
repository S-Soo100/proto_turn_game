## Task B: bird-transform 변칙 룰 연출 개선

### 현재 문제

`BirdTransformEffect.tsx`의 연출이 단순함:
1. `morph`: 돌 위치에 새 아이콘(36px) 등장 + scale 1.2
2. `fly`: scale 1.5 + rotate 30 + y -300 으로 사라짐
3. `feather`: 깃털 아이콘 3개 낙하
4. `message`: "...?" 텍스트

새 이미지(`chaos-bird.png`)가 귀여운 참새 캐릭터인데, 단순 이동만으로는 "날아가는 느낌"이 약함.

### 개선 방향

#### B1. 새 비행 애니메이션 강화

- **morph 단계**: 돌이 떨리다가 → 새로 변신 (scale 0 -> 1 + 회전 + 반짝 파티클)
- **fly 단계**: 직선 이동이 아니라 **포물선 궤적**으로 비행
  - `keyframes`로 곡선 경로: 위로 올라갔다가 → 대각선 우상단으로 날아감
  - 날개짓 효과: `scaleX` 반복으로 좌우 흔들림 (날갯짓 시뮬레이션)
  - 크기 변화: 가까울 때 크고 → 멀어지며 작아짐 (원근감)
- **trail 추가**: 비행 경로에 반짝이 파티클 잔상 (`chaos-sparkle.png` 활용)

#### B2. 필요한 추가 에셋

| 에셋 | 용도 | 생성 필요 |
|---|---|---|
| `chaos-bird.png` | 새 본체 (참새) | 이미 생성됨 |
| `chaos-bird-fly.png` | 날개 편 비행 자세 새 | **신규 생성** |
| `chaos-feather.png` | 깃털 | 이미 생성됨 |
| `chaos-sparkle.png` | 비행 잔상 파티클 | 이미 생성됨 |

- `chaos-bird-fly.png`: 날개를 활짝 편 옆모습 참새, 같은 아트 스타일
  - 256x256, 1:1, transparent background
  - morph에서 `chaos-bird.png` → fly에서 `chaos-bird-fly.png`로 전환

#### B3. 코드 변경 상세

```
BirdTransformEffect.tsx 변경:
1. phase 확장: 'morph' | 'fly' | 'trail' | 'feather' | 'message' | 'done'
2. morph: 돌 이미지 → chaos-bird.png 크로스페이드 (0.3s)
3. fly: chaos-bird-fly.png로 교체 + CSS keyframes 포물선
   - @keyframes birdFlight {
       0%   { top: stoneY; left: stoneX; scale: 1; opacity: 1; }
       30%  { top: stoneY - 40%; left: stoneX + 10%; scale: 1.3; }
       70%  { top: 10%; left: 70%; scale: 0.8; }
       100% { top: -10%; left: 90%; scale: 0.4; opacity: 0; }
     }
   - scaleX(-1) <-> scaleX(1) 반복으로 날갯짓
4. trail: 비행 경로에 sparkle 3~5개 시차 등장 + 페이드아웃
5. feather: 기존 유지 (깃털 3개 낙하)
6. message: "...?" 유지
```
