## 요약

> GonggiBoard.tsx의 게임 루프(타이머, 애니메이션, 상태 전환)를 기획서 기반으로 전면 재편한다. 현재 코드는 T-GG017~T-GG019에 걸친 반복적 핫픽스로 타이머 관리, 상태 전환, 일시정지/재개 로직이 불안정하다.

## 배경 / 현재 문제

### 미해결 버그 2건

#### 버그 1: 돌이 위로 올라가서 사라짐 (25% 빈도)
- **증상**: toss 후 FlyingStone이 화면 상단으로 올라간 뒤 사라지고, catch 불가
- **빈도**: 약 4번 중 1번
- **조건**: 카오스 이벤트 미발동 시 (정상 플레이 중)
- **히스토리**: T-GG017(타이밍), T-GG018(CSS 2단계), T-GG019(RAF 재구현) 3회 시도 실패
- **추정 원인**: `startTossAnimation` RAF 루프와 `handleCatch`/`clearTossTimers` 간의 경합. RAF 콜백이 cancel 전에 실행되어 auto-miss 처리하거나, DOM ref 조작과 React state 갱신 타이밍 불일치

#### 버그 2: 1단 진행 중 "성공" 메시지 후 즉시 실패 처리
- **증상**: 1단 substep 진행 중 (예: 2번째 돌 pick 시점) "성공" 메시지가 나타남 → 이후 아무 동작 없이 실패 처리
- **추정 원인**: `advanceSubstep()` → `catchStone()` → `pickStones()` 체이닝에서 상태 불일치. pick timer와 auto-miss timer가 동시에 만료되거나, `catchStone` 성공 후 `advanceSubstep`이 잘못된 phase로 전환

### 구조적 문제 (코드 분석 결과)

| # | 문제 | 위치 | 영향 |
|---|---|---|---|
| 1 | **Pick timer 일시정지 미지원** | `handlePause()` | `pickTimerRef` 미클리어 → 일시정지 중 자동 실패 |
| 2 | **Chaos timeout 일시정지 미지원** | `handlePause()` | `chaosTimeoutRef` 미클리어 → 일시정지 중 상태 전환 |
| 3 | **RAF 애니메이션 로직 중복** | `startTossAnimation` vs `handleResume` | 물리 수식 2곳 유지, 불일치 위험 |
| 4 | **타이머 4종 분산 관리** | 전체 | elapsed(interval) + pick(interval) + flight(RAF) + chaos(timeout[]) 각각 독립 관리, 정리 누락 위험 |
| 5 | **DOM ref + React state 이중 제어** | FlyingStone | `display: none/block` DOM 직접 조작 + `tossAnimating` state → 동기화 실패 시 유령 렌더링 |
| 6 | **gameStateRef 이중 진실원** | 전체 | `gameState`(React) + `gameStateRef`(ref) 동기화 의존 → 콜백 내 stale state 위험 |

## 재편 전략

### 핵심 원칙
1. **기획서가 진실**: `planning/gonggi/overview.md`의 phase 흐름을 정확히 구현
2. **단일 진실원**: 게임 상태는 `useReducer`로 단일 관리 (useState + ref 이중 구조 폐기)
3. **타이머 중앙 관리**: 모든 타이머(interval, timeout, RAF)를 하나의 관리자에서 통합
4. **일시정지 완전 지원**: pause 시 모든 타이머 정지, resume 시 정확한 잔여 시간으로 재개

### Phase 흐름 (기획서 기준)

```
scatter → select → hold → toss → [pick+catch 병행] → advanceSubstep
                                                        ├→ 다음 substep (select/toss)
                                                        ├→ stage-clear
                                                        └→ round-clear (stage 5)

실패 시: → failed → retrySubstep/retryStage → scatter/select/toss
```

#### Stage별 차이

| Stage | Select | Hold | Toss | Pick | Catch |
|---|---|---|---|---|---|
| 1~4 | ✓ 바닥 돌 선택 | ✓ 손으로 올림 | ✓ 선택 돌 1개 | ✓ 바닥에서 N개 | ✓ 떨어지는 돌 탭 |
| 5 | ✗ (전체 toss) | ✗ | ✓ 전체 5개 | ✗ | ✓ 떨어지는 돌 탭 |

## 구현 계획

### 1. TimerManager 유틸리티

```typescript
// src/lib/game-logic/gonggi-timer.ts (새 파일)
interface ManagedTimer {
  id: string
  type: 'timeout' | 'interval' | 'raf'
  handle: number
  startedAt: number
  duration: number      // timeout/interval 전용
  elapsed: number       // pause 시 누적 경과
  callback: () => void  // timeout 전용: resume 시 잔여 시간으로 재등록
}

class TimerManager {
  private timers: Map<string, ManagedTimer>

  setTimeout(id: string, callback: () => void, delay: number): void
  setInterval(id: string, callback: () => void, interval: number): void
  startRAF(id: string, animate: (elapsed: number) => boolean): void // boolean: continue?

  clear(id: string): void
  clearAll(): void

  pauseAll(): void     // 모든 타이머 정지 + 경과 시간 저장
  resumeAll(): void    // 잔여 시간으로 재등록

  get isPaused(): boolean
}
```

### 3. FlyingStone 애니메이션 리팩토링

```typescript
// RAF 애니메이션을 TimerManager로 통합
// startTossAnimation과 handleResume의 물리 수식을 단일 함수로 추출

function flyParabola(elapsed: number, duration: number): { y: number; scale: number; rotation: number } {
  const t = Math.min(elapsed / duration, 1)
  const y = 62 + (5 - 62) * 4 * t * (1 - t)
  const scale = 1.0 + (0.65 - 1.0) * 4 * t * (1 - t)
  const rotation = t < 0.5
    ? -15 * Math.sin(t * Math.PI)
    : 10 * Math.sin((t - 0.5) * 2 * Math.PI)
  return { y, scale, rotation }
}

// TimerManager.startRAF로 등록 → pause/resume 자동 지원
timerManager.startRAF('flight', (elapsed) => {
  const { y, scale, rotation } = flyParabola(elapsed, duration)
  // DOM 업데이트
  if (elapsed >= duration) {
    dispatch({ type: 'CATCH_MISS' })
    return false // stop RAF
  }
  return true // continue
})
```

## 메모

- `useReducer`로 전환하면 `gameStateRef` 이중 관리가 불필요해짐 — dispatch는 항상 최신 state에서 동작
- TimerManager는 gonggi 전용이지만, 향후 다른 게임에서도 재사용 가능
- RAF 물리 수식은 `flyParabola()` 순수 함수로 추출하여 unit 테스트 가능
- 기존 T-GG017~T-GG019의 변경은 모두 이 티켓에서 대체됨
