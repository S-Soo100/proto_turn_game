## 요약

> FlyingStone CSS 애니메이션을 2단계로 분리: (1) 상승 → 공중 정점에서 체류(catch 대기) (2) catch 안 하면 낙하하면서 miss 처리. 현재 1단계 포물선(바닥→꼭대기→바닥)에서 돌이 바닥으로 복귀하여 유저가 잡을 수 없다고 인식하는 버그를 해결한다.

## 배경 / 현재 문제

### 현재 `flightArc` 애니메이션
```
0%   → top: 62% (바닥)
35%  → top: -4% (꼭대기)
100% → top: 62% (바닥 복귀) ← 문제
```
- 애니메이션 duration (예: 3000ms) 동안 돌이 올라갔다가 다시 바닥으로 내려옴
- 100% 시점에서 돌이 바닥의 다른 돌들과 시각적으로 겹침
- auto-miss는 `duration + 1000`ms에 발동 → 애니메이션 종료 후 1초 동안 돌이 바닥에 머무름
- 유저 입장: "돌이 바닥으로 돌아갔는데 잡을 수 없다" → 버그로 인식

### 원하는 동작
1. 돌이 위로 올라감 (상승)
2. 공중 정점(top: ~10%)에서 **체류** — 살짝 떠다니는 호버 효과
3. 유저가 탭 → catch 성공
4. 탭 안 하면 → 돌이 **낙하** + miss 처리 + "놓쳤어요!"

## 상세 설명

### 1단계: 상승 애니메이션 (`flightUp`)
- 바닥(top: 62%) → 정점(top: 10%) 상승
- duration: `tossDuration * 0.4` (기존 35% 비율과 유사)
- 회전 + 스케일 효과 유지
- 상승 완료 후 → **체류 상태 진입**

### 2단계: 공중 체류 (`hovering`)
- top: 10% 부근에서 미세 상하 진동 (±3% 정도)
- catch-zone 글로우 활성
- 지속 시간: `tossDuration * 0.6` (남은 시간)
- 유저가 탭 → catch 성공 → FlyingStone 제거
- 탭 안 함 → 3단계로 전환

### 3단계: 낙하 + miss (`flightDown`)
- 체류 시간 만료 → 돌이 아래로 빠르게 낙하 (top: 10% → 120%)
- 화면 밖으로 사라짐 (바닥에 돌아오지 않음)
- 낙하 완료 → `catchStone(state, false, 'miss')` → failed phase
- "놓쳤어요!" 메시지

### 타이밍 설계

| 구간 | 비율 | 3000ms 기준 | 동작 |
|---|---|---|---|
| 상승 | 40% | 1200ms | 바닥 → 정점 |
| 체류 | 50% | 1500ms | 정점에서 호버, catch 가능 |
| 낙하 | 10% | 300ms | 정점 → 화면 밖 (miss 시에만) |

auto-miss 타이머: `tossDuration` 시점에 발동 (기존 `duration + 1000` → `duration`으로 변경).
상승(40%) + 체류(50%) = 90% 시점에서 자동 낙하 시작.

### 구현 방법

#### A. 애니메이션 state 추가

```typescript
type FlightPhase = 'ascending' | 'hovering' | 'falling'
const [flightPhase, setFlightPhase] = useState<FlightPhase>('ascending')
```

#### B. 3개 keyframe 정의

```typescript
const flightUp = keyframes`
  0%   { top: 62%; transform: translateX(-50%) scale(1.0) rotate(0deg); }
  100% { top: 10%; transform: translateX(-50%) scale(0.7) rotate(-10deg); }
`

const hoverFloat = keyframes`
  0%   { top: 10%; transform: translateX(-50%) scale(0.7) rotate(-5deg); }
  50%  { top: 7%;  transform: translateX(-50%) scale(0.72) rotate(0deg); }
  100% { top: 10%; transform: translateX(-50%) scale(0.7) rotate(5deg); }
`

const flightDown = keyframes`
  0%   { top: 10%; transform: translateX(-50%) scale(0.7); opacity: 1; }
  100% { top: 120%; transform: translateX(-50%) scale(0.5); opacity: 0; }
`
```

#### C. startTossAnimation 수정

```typescript
const startTossAnimation = useCallback((state: GonggiState) => {
  const duration = getTossDuration(state.stage)
  const ascendTime = duration * 0.4
  const hoverTime = duration * 0.5

  setTossAnimating(true)
  setFlightPhase('ascending')
  setCatchMessage('')

  // Phase 1→2: ascending → hovering
  const hoverTimer = setTimeout(() => {
    setFlightPhase('hovering')
  }, ascendTime)

  // Phase 2→3: hovering timeout → falling (miss)
  const fallTimer = setTimeout(() => {
    const current = gameStateRef.current
    if (current.phase === 'catch' || current.phase === 'pick') {
      setFlightPhase('falling')
      // After fall animation, trigger miss
      setTimeout(() => {
        setCatchMessage('놓쳤어요!')
        const missState = catchStone(...)
        setTossAnimating(false)
        setGameState(missState)
        gameStateRef.current = missState
        setTimeout(() => setCatchMessage(''), 1500)
      }, 300) // fall duration
    }
  }, ascendTime + hoverTime)

  tossTimerRef.current.push(hoverTimer, fallTimer)
}, [])
```

#### D. FlyingStone 렌더링 — flightPhase에 따라 animation 변경

```tsx
<FlyingStone
  style={{
    animationName: flightPhase === 'ascending' ? flightUp.name
      : flightPhase === 'hovering' ? hoverFloat.name
      : flightDown.name,
    animationDuration: flightPhase === 'ascending' ? `${ascendTime}ms`
      : flightPhase === 'hovering' ? '1000ms'
      : '300ms',
    animationIterationCount: flightPhase === 'hovering' ? 'infinite' : '1',
    animationFillMode: 'forwards',
    pointerEvents: gameState.phase === 'catch' ? 'auto' : 'none',
  }}
  onPointerDown={gameState.phase === 'catch' ? handleCatch : undefined}
>
```

## 메모

- `flightPhase` state를 ref로 할지 useState로 할지: 리렌더링이 필요하므로 useState 사용
- hover 중 `animation-iteration-count: infinite`로 계속 떠다니는 효과
- 낙하 시 `top: 120%`로 화면 완전 밖으로 → 바닥 돌과 절대 겹치지 않음
- pick phase(stages 1-4)에서는 돌이 공중에 떠있는 동안 바닥 돌을 pick → 이 동작은 기존과 동일 (startTossAnimation이 pick phase에서도 호출됨)
