## 요약

> 돌멩이의 자아(stone-eyes) 변칙 룰이 실제 게임에서 발동되지 않는 문제를 점검하고 수정한다.
>
> **결과**: `handleChaosEffect`의 `stones-flee` 케이스에서 `setChaosEffect(null)`을 즉시 호출하여 이펙트가 1프레임도 표시되지 않았고, `failSubstep`도 호출하지 않아 돌이 도주해도 게임이 계속 진행됨. StoneEyesEffect를 chaos overlay에 연결하고 `handleStoneEyesComplete` → failSubstep 호출로 수정.

## 상세 설명

사용자가 플레이 중 돌멩이의 자아 룰을 한 번도 본 적이 없다고 보고함. 다음 관점에서 점검이 필요하다:

### 점검 항목

1. **발동 조건 확인**
   - 트리거: `before-pick` — 줍기 직전에 checkChaos가 정상 호출되는지
   - minRound: 1 — R1부터 발동 가능하므로 조건 자체는 관대함
   - `before-pick` 트리거를 사용하는 유일한 룰이므로 경합 문제는 없음
   - canRepeat: true — 반복 발동 가능하므로 triggeredIds 필터에 걸리지 않아야 함

2. **GonggiBoard.tsx에서 before-pick checkChaos 호출 확인**
   - pick phase 진입 시 `checkChaos(state, 'before-pick', ...)` 호출이 실제로 실행되는지
   - 호출 시점이 올바른지 (pick phase 시작 직후)
   - chaos 결과가 있을 때 `handleChaosEffect`로 전달되는지

3. **StoneEyesEffect 컴포넌트**
   - 눈 팝업 + 도주 애니메이션 정상 렌더링
   - 도주 후 게임 상태 반영 (돌 소실 → 실패 처리)

### 가능한 원인 추정

- `before-pick` checkChaos 호출이 코드 경로상 실행되지 않을 수 있음 (조건 분기에 의해 스킵)
- Stage 5(꺾기)에서는 pick phase가 없으므로, stage 1~4에서만 발동 가능 — pick phase 진입이 정상적인지 확인 필요
- handleChaosEffect에서 `stones-flee` 타입 결과 핸들링이 누락되었을 수 있음
