## 요약

> toss→pick→catch 흐름의 공간감/피드백/동작 일관성 문제를 해결. 던진 돌이 보드 밖까지 포물선 비행하고, 줍은 돌이 🤚 손바닥 영역에 팝인되며, 떨어지는 돌을 직접 탭하여 잡는 UX로 전환.

## 해결한 문제

| 문제 | 원인 | 해결 |
|---|---|---|
| 공간감 부재 — 돌이 제자리에서 출렁임만 | `tossArc` keyframe이 translateY(-80px)만 사용 | `flightArc` keyframe: Container 기준 top 62%→-4%→62% 포물선 비행 |
| pick 피드백 부재 — "돌을 확보했다" 느낌 없음 | 줍은 돌이 보드에 반투명 상태로만 표시 | HandArea 컴포넌트: 🤚 + 돌 이모지 popIn 애니메이션 |
| catch 동작 단절 — 스와이프→스와이프→갑자기 버튼 클릭 | CatchButton으로 잡기 | FlyingStone 직접 탭 = catch (pointerDown) |
| toss phase 막힘 버그 — advanceSubstep 후 UI 없음 | HoldStoneOverlay 조건이 `toss && stage===5`만 | 모든 stage의 toss phase에서 HoldStoneOverlay 표시 |
| FlyingStone 이모지 오류 — 항상 🟡 표시 | `selectedStoneId` null일 때 `STONE_EMOJIS[0]` 폴백 | air/tossed 돌의 실제 ID 기반 이모지 표시 |
| hand 돌 보드 중복 — 같은 돌이 overlay + board에 2번 렌더 | toss phase에서 hand 돌이 보드에도 렌더링 | toss phase에서 hand 돌은 HandArea로 통합 표시 |
