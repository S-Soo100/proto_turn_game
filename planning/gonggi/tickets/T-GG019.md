## 요약

> CSS keyframes 기반 FlyingStone 애니메이션을 전면 폐기하고, requestAnimationFrame + 포물선 수식으로 돌 위치를 직접 제어하는 방식으로 재구현한다.

## 배경

CSS animation-name 전환, fill-mode, 타이밍 불일치 등으로 인한 반복적 버그 (T-GG017, T-GG018) 해결 실패. CSS 애니메이션 접근 자체를 폐기한다.

## 원하는 동작

1. 돌을 고른다 (select → hold)
2. 돌이 내 앞으로 나온다 (hold overlay)
3. 돌을 위로 던진다 (swipe/tap)
4. **돌이 화면 상단까지 올라갔다가 떨어진다** (자연스러운 포물선)
5. **돌이 화면 하단에 떨어지기 전까지** 바닥 돌을 잡고(pick), 떨어지는 돌을 탭(catch)

## 구현

### 핵심: requestAnimationFrame + 포물선 수식

```typescript
// t: 0→1 (경과 비율)
// y(t) = startY + (peakY - startY) * 4 * t * (1 - t)
// t=0: 바닥(62%), t=0.5: 정점(5%), t=1: 바닥(62%)
```

- DOM ref로 직접 style.top 갱신 (React 리렌더링 없음)
- CSS keyframes/animation-name/fill-mode 완전 제거
- t=1 도달 시 자동 miss 처리
