## 요약

> 고양이 습격(cat-swipe) 변칙 룰의 연출을 "위에서 기다란 고양이 발이 내려와 왼→오로 보드를 싹 쓸어 돌을 전부 치워버리는" 방식으로 개편한다. 이후 돌이 전부 사라지므로 무조건 다시하기 UI가 표시된다.

## 상세 설명

### 현재 동작
- `CatSwipeEffect`에서 🐾 이모지가 3방향(left/right/top) 중 랜덤으로 등장하여 돌을 밀어냄
- 페이크 "성공!" → 🐾 슬라이드 → 야옹~ 메시지 순서
- trigger: `before-success` (스테이지 클리어 직전)
- 결과: `all-stones-scattered` → 보드 리셋

### 변경 후 동작
1. **페이크 "성공!" 텍스트** (0.4초)
2. **위에서 기다란 직사각형(고양이 발) 등장**: 화면 상단에서 보드 중앙으로 내려옴
3. **왼→오 스와이프**: 직사각형이 보드 왼쪽 끝에서 오른쪽 끝까지 이동하며 모든 돌을 쓸어버림
4. **돌 전부 lost 처리**: 스와이프 완료 시 모든 바닥 돌의 status를 `lost`로 변경
5. **"야옹~" 메시지** + 직사각형 퇴장 (오른쪽으로)
6. **다시하기 UI 표시**: 돌이 전부 없어졌으므로 `failed` phase → 다시 도전 버튼

### 주요 설계 결정
- **방향 고정**: 기존 3방향 랜덤 → **항상 위→아래 등장, 왼→오 스와이프** (직관적)
- **돌 처리**: 기존 `all-stones-scattered`(리셋) → 모든 돌 `lost` 처리 후 `failSubstep()` 호출
- **다시하기**: 돌이 전부 없으므로 자연스럽게 failed → 다시 도전 버튼 노출

## 구현 가이드

### 룰 정의 수정

**파일**: `src/lib/game-logic/chaos-rules/cat-swipe.ts`
- `CatDirection` 타입 제거 (더 이상 방향 랜덤 불필요)
- result type: `'all-stones-lost'` (신규)
- data: 타임라인 단순화 (위→아래 등장 → 좌→우 스와이프 → 퇴장)
- direction 필드 제거

```typescript
export const catSwipeRule: ChaosRule = {
  id: 'cat-swipe',
  name: '고양이 습격',
  trigger: 'before-success',
  minRound: 1,
  probability: 0.35,
  canRepeat: false,
  execute: (_state, _rng) => ({
    type: 'all-stones-lost',
    ruleId: 'cat-swipe',
    animation: 'cat-swipe',
    message: '야옹~',
    data: {
      timeline: [
        { at: 0, action: 'fake-success' },
        { at: 400, action: 'paw-descend' },
        { at: 800, action: 'paw-swipe' },
        { at: 1400, action: 'stones-lost' },
        { at: 1600, action: 'paw-exit' },
        { at: 2000, action: 'message' },
      ],
    },
  }),
}
```

### 이펙트 컴포넌트 개편

**파일**: `src/components/game/chaos/CatSwipeEffect.tsx`

시각 연출:
1. **고양이 발 디자인**: 기다란 직사각형 (`width: 120%`, `height: 60px`)
   - 갈색 배경 + 하단에 🐾 패드 3~4개 배치
   - `border-radius`로 둥근 모서리
2. **Phase 1 — descend**: `translateY(-100%)` → `translateY(40%)` (위→보드 중앙)
3. **Phase 2 — swipe**: `translateX(-120%)` → `translateX(120%)` (좌→우 스와이프)
   - 스와이프 중 돌이 오른쪽으로 밀려나는 모션 (돌 x좌표 증가 → 화면 밖)
4. **Phase 3 — exit**: 오른쪽으로 퇴장
5. **"야옹~" 메시지**: 중앙에 페이드인

### GonggiBoard에서 chaos 결과 처리

**파일**: `src/components/game/GonggiBoard.tsx`

cat-swipe chaos 결과 처리 시:
1. CatSwipeEffect `onComplete` 콜백에서:
   - 모든 floor/hand 돌을 `loseStone()`으로 lost 처리
   - `failSubstep()` 호출 → failed phase 전환
   - → 자연스럽게 "다시 도전" 버튼 표시됨
