## 요약

> 실제 공기놀이에서는 던진 돌을 빨리 잡는 것이 문제가 되지 않으므로, catch phase 진입 즉시 잡기를 허용한다. 또한 떨어지는 돌에 회전 + 시각 효과를 추가하여 낙하를 더 직관적으로 표현한다.

## 배경

현재 문제:
1. **catch 윈도우 제한**: catch phase에서 `catchWindowActive`가 true일 때만 성공 판정. 그 전에 탭하면 "너무 빨라요!" 실패 → 실제 공기놀이와 다름
2. **낙하 시각 단서 부족**: FlyingStone이 top % 이동 + scale 변화만 있어서 "떨어지고 있다"는 느낌이 약함

## 변경 사항

### 1. catch 즉시 허용 — "너무 빨라요" 제거

**현재 흐름**:
```
catch phase 진입
  → catchWindowActive = false (초반)
  → 탭 시 "너무 빨라요!" 실패
  → duration - catchWindow 경과 후 catchWindowActive = true
  → 탭 시 성공
  → duration 경과 → auto-miss
```

**변경 흐름**:
```
catch phase 진입
  → 언제든 탭 = 성공 (perfect)
  → duration 경과 → auto-miss (놓쳤어요!)
```

**수정 코드** (`GonggiBoard.tsx`):

`handleCatch`:
- `if (!catchWindowActive)` early 분기 **제거**
- catch phase이기만 하면 → 항상 `catchStone(state, true, 'perfect')`

`startTossAnimation`:
- `catchWindowActive` 타이머 제거 (더 이상 불필요)
- auto-miss 타이머는 유지

FlyingStone:
- `catch-zone` 클래스 조건: `catchWindowActive` 체크 제거 → catch phase이면 항상 적용
- 초록 글로우가 catch phase 시작부터 표시

`CatchTiming` 타입:
- `'early'` 값은 순수 로직에서 유지 (하위 호환). 단, UI에서 호출하지 않음

### 2. FlyingStone 낙하 시각 효과 강화

**flightArc keyframe 개선** — 하강 구간에 회전 + 약간의 좌우 흔들림 추가:

```css
@keyframes flightArc {
  0%   { top: 62%; transform: translateX(-50%) scale(1.0) rotate(0deg); }
  35%  { top: -4%; transform: translateX(-50%) scale(0.65) rotate(-15deg); }
  40%  { top: -4%; transform: translateX(-50%) scale(0.65) rotate(-10deg); }
  60%  { top: 28%; transform: translateX(-48%) scale(0.78) rotate(20deg); }
  80%  { top: 48%; transform: translateX(-52%) scale(0.88) rotate(-10deg); }
  100% { top: 62%; transform: translateX(-50%) scale(1.0) rotate(0deg); }
}
```

변경 포인트:
- **상승(0→40%)**: 약간의 반시계 회전 (공기 중 흔들림)
- **하강(40→100%)**: 시계→반시계 교차 회전 + translateX 미세 좌우 흔들림
- **착지(100%)**: rotate(0) 복귀

**FlyingStone catch-zone 시각 강화**:
- catch phase 진입 시 즉시 초록 글로우 (기존: catchWindowActive 후에만)
- 하강 중 크기가 커지면서 "다가오는 느낌" (이미 scale 변화로 구현됨)
