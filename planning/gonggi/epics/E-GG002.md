## 목표

> 공기놀이의 물리엔진 파라미터와 게임 흐름을 전면 리워크하여, 실제 공기놀이를 닮은 직관적인 UX를 제공한다.

## 배경

QA 테스트 결과 다음 문제가 발견됨:

1. **돌이 모두 바닥에 가라앉음** — 물리엔진 중력이 활성화되어 scatter 후에도 돌이 화면 하단에 몰림
2. **"던지기" 후 어떻게 해야 하는지 모름** — 돌 하나가 살짝 뛰고 "스와이프하세요" 메시지가 나오지만, 바닥에 붙은 돌과 상호작용 방법이 불분명
3. **돌을 골라서 던지는 과정이 없음** — 실제 공기놀이는 "돌 하나를 골라서 위로 던지고, 떨어지는 동안 바닥 돌을 잡는" 게임인데, 현재는 자동으로 돌이 선택되어 던져짐
4. **"손에 쥐기" 상태가 없음** — 돌을 골라서 손에 올려놓는 시각적 피드백 부재

### 근본 원인 (코드 레벨)

| 문제 | 원인 | 위치 |
|---|---|---|
| 돌이 바닥에 가라앉음 | gravity scale 0.001 (matter.js) — scatter 후에도 중력이 돌을 아래로 끌어내림 | `gonggi-physics.ts` |
| scatter가 안 퍼짐 | scatter force ±0.005 — 중력보다 약해서 실질적 확산 없음 | `gonggi-physics.ts` |
| 던진 돌이 10초간 떠있음 | velocity -10 + gravity 0.001 = 피크까지 10,000ms | `gonggi-physics.ts` |
| 돌을 직접 골라서 던질 수 없음 | handleToss가 startToss()로 자동 선택 → completeToss() 즉시 실행 | `GonggiBoard.tsx` |
| "손에 올리기" 단계 없음 | phase가 scatter→toss→pick→catch만 존재, hold 단계 없음 | `gonggi.ts` |

## 해결 방향 (확정)

### 핵심 결정: matter.js 완전 제거 + CSS 2.5D

- 바닥 = **x,y 평면** → 게임 로직 좌표를 CSS로 직접 렌더링 (물리엔진 없음)
- 위로 던지기 = **z축** → CSS animation으로 높이 시뮬레이션 (scale + shadow + y-offset + opacity)
- matter.js 의존성 완전 제거 → 번들 축소 + 예측 가능한 타이밍

### z축 시각 표현 (CSS 4단서)
| z값 | scale | y-offset | 그림자 | 의미 |
|---|---|---|---|---|
| 0.0 | 100% | 0px | 작고 선명 | 바닥 |
| 0.3 | 90% | -24px | 중간 | 손에 들어올림 |
| 1.0 | 65% | -80px | 크고 흐림 | 최고점 |

### 게임 흐름 리워크
```
scatter → select → hold → toss → pick → catch
              ↑ 신규    ↑ 신규
```
