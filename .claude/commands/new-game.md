사용자가 새 게임의 기획 문서 구조를 생성하려고 한다.

## 입력

$ARGUMENTS

사용자 입력에서 아래 정보를 추출한다:
- **게임 이름** (한국어): 사용자가 말한 게임 이름 (예: "턴 타이밍 게임")
- **폴더 이름** (영어, kebab-case): 게임 이름을 영어 kebab-case로 변환 (예: "turn-timing-game")
- **스코프 코드** (대문자 1~3글자): 폴더 이름에서 자동 생성 (예: "TTG")
- **game_type_id** (영어, kebab-case): 폴더 이름과 동일

정보가 불명확하면 사용자에게 질문한다.

## 수행 작업

### 1단계: 폴더 구조 생성

`planning/{폴더이름}/` 하위에 다음 구조를 생성한다:

```
planning/{폴더이름}/
├── overview.md
├── epics/
├── tickets/
└── usecases/
```

빈 디렉토리에는 `.gitkeep` 파일을 생성한다.

### 2단계: overview.md 생성

`planning/templates/game-overview-template.md`를 기반으로 `planning/{폴더이름}/overview.md`를 생성한다.
- `{게임 이름}` → 사용자가 말한 게임 이름 (한국어)
- `{game_type_id}` → 폴더 이름 (영어 kebab-case)
- 상태는 `research`로 설정
- 사용자가 게임에 대해 설명한 내용이 있으면 "게임 개요" 섹션에 반영
- 에픽 목록의 스코프는 생성한 스코프 코드 사용

### 3단계: 초기 에픽 생성

`planning/templates/epic-template.md`를 기반으로 `planning/{폴더이름}/epics/E-{스코프}001.md`를 생성한다.
- 에픽 제목: `{게임 이름} MVP`
- 게임: `{폴더이름}`
- 상태: `draft`
- 우선순위: `high`
- 작성자: 사용자 (빈칸 유지)
- 생성일/수정일: 오늘 날짜
- 사용자가 게임에 대해 설명한 내용이 있으면 목표/배경/범위 섹션에 반영
- 티켓 목록에 빈 행 3개 추가 (001, 002, 003)

### 4단계: planning/README.md 업데이트

`planning/README.md`의 **스코프 코드 테이블**에 새 게임 행을 추가한다.
- 코드: 생성한 스코프 코드
- 게임/영역: 게임 이름
- 폴더: `planning/{폴더이름}/`

또한 **폴더 구조** 섹션에도 새 게임 폴더를 추가한다.

### 5단계: 결과 보고

생성 완료 후 사용자에게 다음을 보여준다:
1. 생성된 파일/폴더 목록
2. 스코프 코드
3. 다음 단계 안내:
   - "overview.md를 더 자세히 채워주세요"
   - "에픽 E-{스코프}001을 구체화하고 티켓을 만들어주세요"
   - "ready 상태로 바꾸면 구현을 시작할 수 있어요"
